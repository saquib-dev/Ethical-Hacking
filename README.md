## 🗺️ Repository Architecture

To get the most out of this curriculum, it helps to understand how the resources are split:

* **`00-20/` (Core Curriculum)**: Sequential, step-by-step modules designed for linear learning. Each folder contains the primary lessons, theory, and structured milestones for that topic.
* **`learning/` (Deep-Dives & Sandbox)**: Conceptual deep-dives, standalone reference files, and experimental code. Use this directory alongside the numbered modules to reinforce complex topics with practical layouts.
* **`labs/` & `working/`**: Active environments where you will run exercises, write scripts, and test setups.

## 🚀 Getting Started
1. Follow the **Phases 00-20** in order.
2. For each phase, complete the associated **Learning Module** and **Lab**.
3. Master the tools listed in the **Tools Repository** as they appear in the curriculum.
4. Track your progress in the `working/roadmap.md` file.
