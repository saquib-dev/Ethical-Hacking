# Lab 1: Report Writing

## Goal: Learn how to report a bug professionally.

## A good report must have:
1. **Title**: Clear and concise (e.g., "SQL Injection on /login page").
2. **Severity**: Low, Medium, High, or Critical.
3. **Description**: What is the bug?
4. **Steps to Reproduce**: A step-by-step guide so the company can see the bug themselves.
5. **Impact**: What can a hacker do with this bug? (e.g., "I can steal all user emails").
6. **Suggested Fix**: How should the company fix it?

## Task:
Pick one of the labs you did in `04_Practical_Exploitation` (like SQLi) and write a mock report for it. Use the format above.

## Success Check:
- [ ] Report contains all 6 sections.
- [ ] Steps to reproduce are clear enough for a stranger to follow.
