# BloodHound - Practical Lab

## Objective
Collect Active Directory structure data using SharpHound, import it into BloodHound, and identify an attack path to the Domain Admins group.

## Environment
- **Attacker Machine:** Windows 10 VM joined to `LAB.LOCAL` domain.
- **Target:** `LAB.LOCAL` Domain Controller.

## Scenario Steps

### Step 1: Data Collection
1. On the compromised Windows 10 machine, open a command prompt.
2. Execute SharpHound to collect all AD data.
   ```cmd
   SharpHound.exe -c All
   ```
3. SharpHound will generate a ZIP file (e.g., `20231025143000_BloodHound.zip`) containing JSON files.

### Step 2: Database Setup
1. On your analysis machine (e.g., Kali Linux), ensure Neo4j is running: `sudo neo4j start`.
2. Launch the BloodHound GUI.
3. Log in with your Neo4j credentials.

### Step 3: Data Import
1. In the BloodHound GUI, click on the "Upload Data" button (or drag and drop).
2. Select the `.zip` file generated by SharpHound in Step 1.
3. Wait for the data to process and ingest into the Neo4j database.

### Step 4: Analysis
1. Navigate to the "Queries" tab in BloodHound.
2. Click on the pre-built query: **"Find Shortest Paths to Domain Admins"**.
3. Analyze the graph:
   - Identify your compromised starting user.
   - Follow the edges to understand how to escalate privileges (e.g., `User A` -> `MemberOf` -> `HelpDesk Group` -> `ForceChangePassword` -> `User B` -> `MemberOf` -> `Domain Admins`).
4. Document the exact steps required to execute the attack path in the live environment.
