# Hashcat Overview

## What is Hashcat?
Hashcat is widely recognized as the world's fastest and most advanced password recovery utility. Its defining feature is its ability to heavily utilize Graphics Processing Units (GPUs) via OpenCL or CUDA to perform massively parallel hash computations, making it exponentially faster than CPU-bound crackers for complex algorithms.

## How it Works
Similar to John the Ripper, Hashcat takes candidate passwords, hashes them, and compares the output to the target hashes. The difference lies in Hashcat's low-level optimizations and parallel processing architecture.

## Key Concepts
- **Hash Types (`-m`):** Hashcat strictly requires you to specify the exact hash algorithm. It does not auto-detect hashes. You must look up the correct numeric code.
- **Attack Modes (`-a`):** Determines how candidate passwords are generated (e.g., from a list, generated by a pattern/mask, or a combination).
- **Mask Attacks:** Hashcat uses "masks" (e.g., `?u?l?l?l?d?d?d?s`) to define specific character sets for specific positions in a password. This is vastly more efficient than pure, dumb brute-forcing because it targets known password creation patterns (like "Capital letter, lowercase letters, numbers, symbol").

## Best Practices
- **Use GPUs:** Always run Hashcat on a system with dedicated, powerful GPUs for any serious cracking task.
- **Rules are King:** A well-crafted rule set (like `Dive` or `OneRuleToRuleThemAll`) applied to a good dictionary is usually the most effective method for cracking real-world human-created passwords.
- **Temperature Monitoring:** Hashcat pushes hardware to its absolute limits. Ensure proper cooling, and use Hashcat's built-in temperature abortion limits (`--hwmon-temp-abort`) to prevent hardware damage.
- **Session Management:** Like JtR, Hashcat saves sessions. Use the `--session` flag to name your sessions and `--restore` to resume them.
